サービス層を導入したら、一覧ページの使い勝手を高めていきます。本章では検索ボックスや読了フィルタ、並び替えを追加し、少し実用的な一覧に育てます。

ゴールは、タイトル・著者・読了状態・登録日の並び順を条件に IBookService へ問い合わせ、UI から条件を入力して結果を表示できるようにすることです。

## この章で学ぶこと

| 項目 | ポイント |
| --- | --- |

## 検索条件の入力 UI

一覧ページ上部に検索フォームを配置し、条件をバインドするモデル（例：BookSearchCondition）を用意します。

- タイトル/著者：InputText で部分一致検索用の文字列を入力
- 読了フィルタ：All/Read/Unread を選択するドロップダウン
- 並び順：作成日の昇順・降順を選択するドロップダウン
- 「検索」ボタンでサービス呼び出しを実行

## サービス層でのフィルタとソート

IBookService の `GetListAsync` に検索条件を渡し、LINQ でクエリを組み立てます。

```csharp
public async Task<List<Book>> GetListAsync(BookSearchCondition? condition)
{
    var query = _db.Books.AsQueryable();

    if (!string.IsNullOrWhiteSpace(condition?.Title))
    {
        query = query.Where(b => b.Title.Contains(condition.Title));
    }

    if (!string.IsNullOrWhiteSpace(condition?.Author))
    {
        query = query.Where(b => b.Author!.Contains(condition.Author));
    }

    if (condition?.ReadFilter == ReadFilter.Read)
    {
        query = query.Where(b => b.IsRead);
    }
    else if (condition?.ReadFilter == ReadFilter.Unread)
    {
        query = query.Where(b => !b.IsRead);
    }

    query = condition?.SortOrder == SortOrder.CreatedAtAsc
        ? query.OrderBy(b => b.CreatedAt)
        : query.OrderByDescending(b => b.CreatedAt);

    return await query.ToListAsync();
}
```

## BookListRow コンポーネントで 1 行を部品化

テーブルの 1 行を別コンポーネントに分けると、行クリックで編集ページに遷移させたり、行ごとにボタンを追加したりする拡張が容易になります。`<BookListRow Book="book" OnEdit="..." />` のように受け渡し、一覧側は `foreach` の中をすっきりさせましょう。
