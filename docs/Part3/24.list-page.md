---
title: "一覧ページを作成しよう（READ）"
---

本章では、/books に書籍の一覧を表示するページを作成します。まずは検索や並び替えを入れず、単純な読み取りに集中してデータ取得とテーブル表示の流れを確認します。

ゴールは、BooksList.razor を用意し、AppDbContext（後でサービス層に差し替える前提）から取得した Book をテーブルで表示できるようにすることです。

## この章で学ぶこと

| 項目 | ポイント |
| --- | --- |
| ルーティング | `/books` に対応する Razor コンポーネントを定義する |
| データ取得 | DbContext から書籍を取得し、メモリ上のリストに展開する |
| テーブル表示 | `foreach` を用いたシンプルなテーブル描画 |

## BooksList.razor の役割とルーティング

一覧ページはルート `/books` に紐付けます。コンポーネントの先頭で `@page "/books"` を宣言し、ページ遷移の入口を用意します。

```razor
@page "/books"
@inject AppDbContext Db

<h3>書籍一覧</h3>
@if (books is null)
{
    <p>読み込み中です...</p>
}
else if (!books.Any())
{
    <p>まだ書籍が登録されていません。</p>
}
else
{
    <table class="table">
        <thead>
            <tr><th>タイトル</th><th>著者</th><th>読了</th><th>作成日</th></tr>
        </thead>
        <tbody>
            @foreach (var book in books)
            {
                <tr>
                    <td>@book.Title</td>
                    <td>@book.Author</td>
                    <td>@(book.IsRead ? "済" : "未")</td>
                    <td>@book.CreatedAt.ToString("yyyy-MM-dd")</td>
                </tr>
            }
        </tbody>
    </table>
}

 @code {
     private List<Book>? books;

     protected override async Task OnInitializedAsync()
     {
         books = await Db.Books.OrderByDescending(b => b.CreatedAt).ToListAsync();
     }
 }
 ```

 まずは非同期で一覧を読み込み、テーブルに反映するシンプルな構成とします。データが空の場合のメッセージも用意しておくと状態が分かりやすくなります。

初回の読み込みに合わせてデータを取得したいので、ライフサイクルメソッドは `OnInitializedAsync` を使います。ユーザー操作を待たずに最初の状態を整えたいときに適したタイミングです。

## ここではシンプルな READ に絞る

検索やフィルタを先に入れたくなりますが、最初は単一の読み取りに絞るほうが流れを把握しやすくなります。ここでは簡便のため UI 層から直接 `DbContext` を参照していますが、後の章で責務分離のためにサービス層へ置き換える予定です。データ取得部分を分離しやすい形で書いておきましょう。

## 動作確認のポイント

- /books に遷移してテーブルが表示されるか
- データがないときにメッセージが表示されるか
- サンプルデータを手動で投入して並び順が意図どおりか

ここまで完成すれば、READ 機能の最小セットが整いました。次章では CREATE を追加し、データが増えると一覧の動きがどう変わるかを確認します。
