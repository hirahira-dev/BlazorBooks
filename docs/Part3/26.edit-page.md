---
title: "編集・削除を実装しよう（UPDATE/DELETE）"
---

本章では、既存の書籍を編集・削除できるページを実装します。RouteParameter から ID を受け取り、対象の Book を読み込んでフォームに表示し、更新と削除を行えるようにします。

ゴールは、`/books/edit/{id:int}` で編集ページを開き、保存で更新、削除ボタンでレコードを削除し一覧へ戻る流れを整えることです。

## この章で学ぶこと

| 項目 | ポイント |
| --- | --- |
| ルーティング | `/books/edit/{id:int}` のルートパラメーターを受け取る |
| データ読み込み | ID から Book を取得し、フォームに初期値を反映する |
| 更新処理 | 入力値を保存し、一覧へナビゲーションする |
| 削除処理 | confirm ダイアログで確認してから削除する |

## 編集ページの基本構成

`@page "/books/edit/{Id:int}"` でルートを設定し、`[Parameter] public int Id { get; set; }` でパラメーターを受け取ります。取得した Book を EditForm にバインドし、OnValidSubmit で更新します。

```razor
@page "/books/edit/{Id:int}"
@inject AppDbContext Db
@inject NavigationManager Nav

<h3>書籍の編集</h3>
@if (book is null)
{
    <p>対象の書籍が見つかりませんでした。</p>
}
else
{
    <EditForm Model="book" OnValidSubmit="HandleUpdate">
        <DataAnnotationsValidator />
        <ValidationSummary />
        <!-- フィールドは新規作成と同様 -->
        <button class="btn btn-primary" type="submit">保存</button>
        <button class="btn btn-outline-danger ms-2" type="button" @onclick="HandleDelete">削除</button>
    </EditForm>
}

@code {
    [Parameter] public int Id { get; set; }
    private Book? book;

    protected override async Task OnInitializedAsync()
    {
        book = await Db.Books.FindAsync(Id);
    }

    private async Task HandleUpdate()
    {
        await Db.SaveChangesAsync();
        Nav.NavigateTo("/books");
    }

    private async Task HandleDelete()
    {
        if (book is null) return;
        if (!await JsConfirm("本当に削除しますか？")) return;
        Db.Books.Remove(book);
        await Db.SaveChangesAsync();
        Nav.NavigateTo("/books");
    }

    private async Task<bool> JsConfirm(string message)
        => await JS.InvokeAsync<bool>("confirm", message);

    [Inject] IJSRuntime JS { get; set; } = default!;
}
```

削除前にブラウザーの confirm を利用すると、手軽に誤操作を防止できます。サンプルでは `IJSRuntime` を注入して `confirm` 関数を呼び出しています。

## フォームの重複に気づく

新規作成と編集で同じ入力項目が増えるにつれ、フォーム部分の重複が目立ってきます。ここでは一旦そのまま実装し、次章以降でフォーム共通化を検討する伏線として意識しておきましょう。
