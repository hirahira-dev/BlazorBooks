title: "新規作成ページを作ろう（CREATE）"

本章では、新規作成ページ（BookCreate.razor）を追加し、フォームから書籍を登録できるようにします。入力とバリデーション、保存後のナビゲーションまでを一通り体験します。

ゴールは、EditForm と DataAnnotations を組み合わせ、入力値を Book モデルにバインドして保存し、成功時に /books へ戻る流れを完成させることです。

## この章で学ぶこと

| 項目 | ポイント |
| --- | --- |
| ルーティング | `/books/create` へのページ設定 |
| フォーム構築 | EditForm と InputText などの基本コンポーネントの使い方 |
| バリデーション | DataAnnotations と ValidationSummary の連携 |
| 保存処理 | データ登録後に一覧へナビゲーションする流れ |

## BookCreate.razor の構成

ルートを `/books/create` に設定し、Book のインスタンスをフォームとバインドします。DataAnnotations を付与した Book をそのままモデルに使うと、バリデーションを一箇所に集約できます。

```razor
@page "/books/create"
@inject AppDbContext Db
@inject NavigationManager Nav

<h3>書籍の新規登録</h3>
<EditForm Model="book" OnValidSubmit="HandleSubmit">
    <DataAnnotationsValidator />
    <ValidationSummary />
    <div class="mb-3">
        <label class="form-label">タイトル</label>
        <InputText class="form-control" @bind-Value="book.Title" />
    </div>
    <div class="mb-3">
        <label class="form-label">著者</label>
        <InputText class="form-control" @bind-Value="book.Author" />
    </div>
    <div class="mb-3">
        <label class="form-label">メモ</label>
        <InputTextArea class="form-control" @bind-Value="book.Memo" />
    </div>
    <div class="form-check mb-3">
        <InputCheckbox class="form-check-input" @bind-Value="book.IsRead" />
        <label class="form-check-label">読了済み</label>
    </div>
    <button class="btn btn-primary" type="submit">保存</button>
</EditForm>

@code {
    private Book book = new();

    private async Task HandleSubmit()
    {
        book.CreatedAt = DateTime.Now;
        Db.Books.Add(book);
        await Db.SaveChangesAsync();
        Nav.NavigateTo("/books");
    }
}
```

`@bind` を使うことで、フォーム入力が直接 Book インスタンスに反映されます。OnValidSubmit でのみ保存処理を行うため、バリデーションエラーがある場合は保存されません。

現時点では作成日時 `CreatedAt` を UI 層でセットしていますが、後でサービス層を導入した際に共通ロジックとして移動させる予定です。役割を分ける下準備として意識しておきましょう。

`OnSubmit` はバリデーション結果に関わらず常に発火するのに対し、`OnValidSubmit` はすべての検証が通った場合にだけ実行されます。必須項目が揃ってから保存したいときは `OnValidSubmit` を選び、独自の検証ロジックを実装したい場合は `OnSubmit` でハンドリングするといった使い分けができます。

## よくあるつまずき

- **バリデーションが動かない**：`<DataAnnotationsValidator />` を配置しているか、`EditForm` の `Model` に DataAnnotations を付与した型を設定しているかを確認します。
- **ボタンを押しても OnValidSubmit が呼ばれない**：`type="submit"` が付いているか、`EditForm` の外にボタンを置いていないかを見直します。
- **保存後に画面が遷移しない**：`NavigationManager` のインスタンスが注入されているか、`NavigateTo` のパスが正しいかを確認します。
