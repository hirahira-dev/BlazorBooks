本章では、書籍管理アプリの土台となるプロジェクトとデータモデルを整えます。ここで環境を固めておくことで、後続の UI 実装やサービス層導入をスムーズに進められます。

目標は、Blazor Web App（サーバーインタラクティブ）を作成し、Book エンティティと AppDbContext、SQLite の設定、マイグレーションまでを準備することです。UI はまだ最低限で構いません。

## この章で学ぶこと

| 項目 | ポイント |
| --- | --- |
| プロジェクト作成 | Blazor Web App（InteractiveServer）テンプレートの作成手順 |
| データモデル設計 | Book エンティティのプロパティと役割 |
| データベース設定 | AppDbContext と SQLite の設定、マイグレーションの流れ |

## プロジェクトの準備

1. `dotnet new blazor` でサーバーインタラクティブのプロジェクトを作成します。
2. Entity Framework Core の SQLite プロバイダーを追加します。
3. `Program.cs` で `AddDbContext<AppDbContext>(options => options.UseSqlite("Data Source=app.db"));` を設定し、DI に登録します。

このテンプレートでは、サーバー側で UI をレンダリングし、必要に応じてシグナル経由で双方向通信する InteractiveServer モードが既定で
有効になります。最初のうちは「レンダリングはサーバー、ブラウザーは表示とイベント送信を担当する」くらいに捉えておけば十分です。

この時点ではコンポーネント構成やレイアウトはテンプレートのままで問題ありません。まずはデータベース関連の基礎を固めましょう。

## Book エンティティの設計

書籍を表す Book クラスには次のプロパティを定義します。

- `Id`：主キー
- `Title`：必須、最大 100 文字
- `Author`：任意、最大 100 文字
- `Memo`：任意、最大 1000 文字
- `IsRead`：読了フラグ
- `CreatedAt`：作成日時（新規作成時に現在時刻を設定）

設計のポイントは、入力制約を DataAnnotations で表現しておくことです。後続のフォームバリデーションと整合します。

DataAnnotations をモデル側に置くことで、クライアント・サーバー間で同じルールを共有できます。これにより、編集フォームでも API
でも一貫した検証が働き、バリデーションの定義を二重に持つ必要がなくなります。

## AppDbContext と SQLite 設定

データアクセスのハブとして AppDbContext を用意し、Book エンティティを DbSet で公開します。

```csharp
public class AppDbContext : DbContext
{
    public DbSet<Book> Books => Set<Book>();

    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options)
    {
    }
}
```

SQLite を利用するため、接続文字列はシンプルに `Data Source=app.db` とします。開発環境ではファイルベースのデータベースで十分です。

## マイグレーションの流れ

1. `dotnet ef migrations add InitialCreate` で初回マイグレーションを作成します。
2. `dotnet ef database update` で SQLite ファイルを生成し、テーブルを適用します。

マイグレーションを適用したら、`app.db` が作成されていることを確認しましょう。ここまででデータストアの準備が完了し、次章で一覧 UI を作る準備が整います。
